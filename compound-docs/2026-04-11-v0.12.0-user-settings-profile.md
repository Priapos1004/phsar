# v0.12.0 — User Settings, Theme System, Admin & Account Management

**Date:** 2026-04-11 | **Branch:** v0.12.0-user-settings-profile

## Summary

Added per-user settings (theme, name language, rating step, default search view, spoiler level), a CSS custom property theme system with `@property` indirection for Tailwind compatibility, a spoiler frontier algorithm with precomputed visibility cache (three modes: off/blur/hide), admin page for registration token management, user registration page, account deletion with glass-crack UX, and flat media-level data export. Guest account support via optional env vars.

## Key Decisions

- **`@property` + `var()` indirection for themes**: Tailwind 4's `@theme inline` inlines static values into utilities at build time. Wrapping values in `var(--primary)` forces Tailwind to emit the `var()` reference, enabling runtime theme switching via `.theme-*` classes on `<html>`. Without this, theme overrides would have no effect.
- **Theme = unified setting (not separate color + avatar)**: Each theme controls primary color, gradient, chart palette, AND hero character pic as one selection. Simpler UX and avoids invalid combinations.
- **Flat media-level export**: One row per media (not per-anime or separate ratings/watchlist sections) makes the data analysis-ready for spreadsheet users. Conditional name columns avoid empty columns when user uses romaji.
- **`ondelete="SET NULL"` on registration token FKs**: Tokens survive user deletion. `used_at` timestamp (not the FK) determines "used" status, so `[deleted]` users are handled gracefully in the admin token list.
- **Module-level `RATING_ATTRIBUTE_FIELDS`**: Moved off the SQLAlchemy model class to avoid potential mapper interference. Referenced by export service for column generation.
- **Spoiler frontier precomputed cache**: `user_visible_media` table stores visible media IDs per user. Recomputed per-anime on rating changes (bounded work) and per-user on registration (one-shot). Hide-mode searches filter via `WHERE media.id IN (...)` against the cache — cheap server-side filter, no post-processing.
- **Frontier = last rated main + 1, plus explicitly rated entries**: Per anime, visible media are everything up to and including the first unrated main-story entry after the last rated main. Individually rated media (e.g., an OVA rated out of order) stay visible regardless of frontier. Unrated users see the first main-story entry only.
- **Anime-level content is never spoiler-protected**: Only episode-level media hide; anime covers, titles, and descriptions stay visible so users can still navigate to unwatched content. Intentional asymmetry between anime and media views.

## Failed Approaches

- Tried CSS variable-based chart colors via `var(--color-chart-1)` directly in ECharts — ECharts doesn't resolve CSS variables. Must read computed values via `getComputedStyle` and pass raw color strings.

## Gotchas & Learnings

- **Token reuse after account deletion**: Registration checked `was_used_for_user_id` (FK) to determine if a token was used, but `ondelete="SET NULL"` nullifies that FK on user deletion — allowing the token to be reused. Fix: check `used_at` timestamp instead, which is immutable. The admin service already used `used_at` correctly; the registration path did not. Always use immutable fields for security-critical checks when FKs have cascade behavior.
- `classList.forEach` + `classList.remove` inside the loop mutates the DOMTokenList during iteration — browsers handle it but it's fragile. Safer to collect-then-remove.
- FOUC prevention inline script runs before Svelte hydrates — must have its own allowlist validation since the layout's `isValidTheme()` hasn't run yet.
- `dict.setdefault()` appends keys at the end of Python dicts, breaking logical column ordering in CSV. Solved with explicit canonical `_COLUMN_ORDER` list.
- Pydantic 422 validation errors return `detail` as an array of objects, not a string — frontend `handleResponse` must handle both shapes to avoid `[object Object]` display.
- **Safari leaves blur teardown artifacts**: Removing `filter: blur()` in Safari leaves faint horizontal lines across revealed content. Fix in `SpoilerGuard.svelte`: keep the blurred wrapper mounted after reveal and apply `transform: translateZ(0)` to pin a persistent GPU compositor layer, so the blur layer transitions out instead of being discarded mid-frame. Applied unconditionally — `translateZ(0)` is safe in other engines.
- **FastAPI validates path params before dependencies**: Tests that assert 403 on a route with a typed path param (e.g., `uuid: UUID`) must pass a syntactically valid value — otherwise FastAPI returns 422 for the bad path param before the auth dependency fires and the test never reaches the authorization check.

## Future Work

- Export builds entire payload in memory — acceptable for a personal app with few users, but should switch to `StreamingResponse` if data volumes grow.
- `create_default_settings` relies on caller to commit — all current callers do, but the implicit contract is fragile. Consider adding explicit commit or docstring.
- Token expiry timer can fire during account deletion farewell dialog — edge case, minor UX disruption. Could clear timer on deletion success.
- Refactor `create_user_with_role` test fixture to return both JWT and registration token string — would simplify tests that need the token (e.g., token-reuse and token-preservation tests).
