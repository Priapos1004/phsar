# v0.14.12 — Ratings page (+ statistics)

**Branch:** v0.14.12-ratings-page · **Issue:** #40 "Statistics page" / design-doc Feature 9

## Summary

Built the real `/ratings` page (was a navbar placeholder). Tabbed route (**Ratings** +
**Statistics**, `?tab=` like `/admin`) driven **entirely client-side from one extended
`GET /ratings/scores` fetch** shared by both tabs. Ratings tab: anime-level list with a
score-band cover grid (default) + a sortable table, plus genre/age/season filters. Stats
tab: summary, score histogram, You-vs-MAL alignment, genre/studio breakdown, attribute
correlations, and activity charts. Removed the dead `/statistics` navbar placeholder.

## Key decisions

- **One fetch, two consumers — extend the existing projection, don't add an endpoint.**
  The whole page derives from `RatingScoreItem` (already feeding the rating-consistency
  helper). Added fields (`anime_cover_image`, `mal_score`, `scored_by`, `episodes`,
  `total_watch_time`, `anime_season_name/_year`, `created_at`, `relation_type`) are **all
  columns/hybrids on the already-eager-loaded `media → anime` rows, so zero extra query
  cost** — the DAO query is unchanged. No per-user stats endpoint, **no scipy** (weighted
  least-squares, Spearman ρ, eta are closed-form TS in `ratingStats.ts`).
- **Stats tab gated `{#if active==='stats'}` (remounts on entry), inner sections are real
  tabs** (only the active one mounts) → each chart replays its entrance animation on entry
  and we never hold ~6 chart instances live. Active section persists in the `ratingsFilter`
  store (not the URL — `?tab=` owns only the top-level tab; same call as `adminJobsFilter`).
- **Anime score = simple mean of rated media** (main-chain weighting deferred). Grid bands
  10…0 with fixed within-band order (rating desc, title asc); the grid arrow flips only
  section order, the table sorts per-column — two ordering concepts coexist in one store.
- **Filters act per-media *before* grouping**: an anime survives if ≥1 rated media matches;
  its score/status/counts then reflect only the matching subset (intentional — confirmed).
- **Attributes use correlation, not averages** (averaging ordinal/nominal codes is
  meaningless): quality scales → signed Spearman ρ reframed as importance `|ρ|`; categorical
  choices → per-category mean vs overall + sample-weighted eta. A test asserts the two key
  lists partition every rated attribute (a new attribute can't vanish from both charts).

## Gotchas & learnings

- **⭐ Chart entrance animations never replayed — fixed generically in `EChart`.** ECharts'
  `ResizeObserver` fires an initial callback right after `observe()` that landed mid-animation
  and snapped bars to final state (and `init` could read a 0-width pre-layout). Fix: hold the
  first `setOption` until the first measured ResizeObserver callback, guarded on
  `clientWidth > 0` so a chart mounted hidden still animates when shown. One shared-component
  change → every chart animates on (re)mount; the `{#if}` remount supplies the fresh mount.
- **ECharts tooltip `formatter` returns HTML (`innerHTML`) — Svelte auto-escaping doesn't
  reach inside it.** Any catalog-sourced string (titles, genre/studio names) spliced into a
  formatter must go through `escapeHtml` (`formatString.ts`); enum labels/numbers/ECharts
  `marker` spans are safe. Four sinks hit: the tag, alignment, and activity-trend tooltips,
  **plus the pre-existing `RatingsOverviewTimeline`** (same bug already shipped on `main`).
- **Nullable-column sort needs the comparator to own its direction.** `cmpNullable` returned
  a direction-aware null sentinel that `sortAnimeRows` then re-multiplied by the sort `sign` —
  double-signing scattered null-MAL rows through the column and broke the non-null order.
  Fix: nulls always return ±1 (always last), real values resolve direction *inside*
  `cmpNullable`, and that result is NOT re-signed; the other columns keep the outer `*sign`.
- **Client time math: compare on epoch ms, not the raw ISO string, and drop unparseable
  dates.** A `created_at` `>=` string compare against a JS-built cutoff silently mis-clips if
  backend serialization differs (Z vs +00:00, precision); a malformed date `getTime()`s to
  `NaN` and corrupts the running total / time axis. `epochMs()` normalizes both.
- **Theme ECharts tooltips via the shared `chartTooltipStyle`** (primary-tinted surface,
  `confine:true`) so chart hovers match the app `Tooltip` — three tooltip types now exist
  (app `Tooltip`, ECharts chart tooltip, native `title=`), documented in `frontend/CLAUDE.md`.
- **HTML y-axis labels for tooltips/links on ticks** (Genres & Studios chart): ECharts canvas
  labels can't carry the app `Tooltip` or links, so labels render as HTML aligned to bars via
  shared `ROW`/`TOP_PAD`/`AXIS_PAD` constants. Trade-off: drifts a few px at extreme zoom;
  `yAxis.triggerEvent` is the robust native fallback. With `containLabel:false` you must
  reserve `grid.bottom/left/right` or ticks vanish/clip.
- **Light cards on a dark page is a recurring trap.** `--color-card` is white; text inside a
  card is `card-foreground`/`muted-foreground`, never the page `foreground` or `text-white/*`.
- **Genre-dropdown clipping was a stacking-context issue**, not overflow: the results table is
  its own context (`backdrop-blur`) and painted over the dropdown — needs `overflow-visible`
  on the filter card AND `relative z-20` on the filter wrapper.
- **One shared `SEASON_ORDER`** (`getSeason.ts`) now feeds the spoiler frontier + the ratings
  season filter (was two divergent copies, 0- vs 1-based); mirrors backend `filter_service`.
- **`searchByStudio` (`navigation.ts`)** is the single studio-search jump used by `StudioLinks`
  *and* the tag chart, replacing two byte-identical inline `navigateToSearch({...})` copies.
- Svelte parse gotcha: `$derived(a ?? b || c)` won't compile (`??`/`||` need parens) — use
  `$derived.by(() => …)`. Extending `RatingScoreItem` means updating BOTH test factories
  (`rating-neighbors` + `rating-stats`) or the suite won't typecheck.

## Future work / technical debt

- **`RatingScoreItem` is now a wide DTO** (cover, mal_score, scored_by, episodes, watch time,
  season, relation_type, …) feeding both the consistency helper and the whole stats page.
  Decision point: split if it grows much further or needs stats-only heavy fields.
- **Re-add the You-vs-MAL "Furthest from your trend" list** once the linear fit is predictive
  (gate on R²); removed for now — residual rankings mislead at R²≈0.4.
- **Restricted (guest) users reach `/ratings`** but can't rate, so it just empty-states. No
  role gate added (out of scope); revisit if guests should be redirected.
- Deferred: true watch-history time series (watch_event + rewatches; v1 uses the `created_at`
  proxy), taste-cluster analytics (#40), main-chain-weighted anime score.
- Cosmetic drift (documented inline, low): `AGE_RATING_LABELS`↔backend `AGE_RATING_MAP`
  (has a `?? age` numeric fallback), `scoreColor` bands ↔ `BAND_WORD`, HTML-label↔bar geometry.
