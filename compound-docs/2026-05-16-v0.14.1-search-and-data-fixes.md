# v0.14.1 — Search & data fixes (relation classifier redesign)

**Date:** 2026-05-12 → 2026-05-16 | **Branch:** v0.14.1-search-and-data-fixes (18 commits) | **Status:** ready to merge.

## Summary

Replaces the inline relation-classification baked into the BFS with a two-pass model: BFS captures edges only (no classification), then a pure-function `classify_anime_relations(nodes, edges) -> (classifications, anchor)` runs at scrape, merge-survivor, and lifespan-backfill sites. Adds an `AlternativeVersion` enum value for retellings (Evangelion Rebuild, Hokuto no Ken alts), threads it through the spoiler frontier as a co-anchor, and persists raw MAL relations in a `MediaRelationEdges` 1:1 sidecar so bridge edges across previously-different-anime boundaries activate when split franchises later merge. Companion fixes: anime-view search filters now mirror the anime-card derivation (max age rating, priority-collapsed airing status), title search ranks substring + pg_trgm matches above raw cosine distance, the seeder strips season suffixes from Anime umbrella titles, merge detection now covers new × new pairs at scrape time and is admin-re-triggerable, and `update_sweep` step 1 backfills `MediaRelationEdges` from the bundled `/anime/{id}/full` relations block at zero extra MAL cost.

## Key Decisions

- **Sidecar table, not inline JSONB on `Media`.** Raw MAL edge lists are read only at merge / preview / backfill / sweep-step1 time. Keeping them in a 1:1 `MediaRelationEdges` row off the canonical `Media` row means `selectinload(Anime.media)` on detail / search hot paths doesn't drag the JSONB through every load.
- **Edges persisted UNFILTERED (including dangling targets).** Originally we filtered edges whose target wasn't in the local catalog. That dropped bridge edges that would activate later — split-franchise scenario: Dr. Stone S1 and S3 scraped, then S2 scraped separately, bridge edge S1↔S2 sat dangling and was lost. Now persist everything; `_build_adjacency` filters dangling endpoints defensively at adjacency-build time, so merge consolidation surfaces the previously-dangling bridge.
- **`AlternativeVersion` enum + spoiler-frontier co-anchor.** Retellings extend the story (Eva TV → Rebuild Movies 1-4 each tell the same arc in a new form), so the frontier treats `main` AND `alternative_version` as anchors. Rating Eva TV reveals Rebuild Movie 1; rating Movie 1 reveals Movie 2; etc. Default would've been `side_story`, which would have leaked all four Rebuild movies as soon as TV was rated.
- **Substance gate accepts NULL episodes for currently-airing.** TV/ONA needs `episodes>=8 AND duration>=900s`, but NULL episodes is the legitimate state for a show that's actively airing — MAL only fills the field once the season concludes. Treating NULL as "fails the gate" would have demoted every just-started TV anchor to side_story for weeks.
- **Anchor tier: TV(1) > ONA(2) > Movie(3) > other(4); within-tier oldest aired_from.** Necessary for donghua/Chinese animation whose canonical main is often an ONA, and for the pilot-aired-before-the-real-show edge case where a Movie predates the TV.
- **7-umbrella-field drift detection, embedding regen only on title-affecting drift.** `reclassify_anime` rewrites mal_id, title, name_eng, name_jap, other_names, description, cover_image on any drift, but regenerates the AnimeSearch embedding only when title/name fields shifted. Bare description tweaks or cover-image updates don't burn the model invocation.
- **Per-anime commit in `relation_backfiller`.** Cold start lazy-fetches missing sidecars from MAL at 1 req/s — ~14 min for an 800-media catalog. Per-anime checkpointing means a crash 12 min in keeps 11 min of work; without it, a single kill loses the whole batch.
- **Sweep step 1 also refreshes `MediaRelationEdges`** from the bundled `/anime/{id}/full` `relations` block — zero extra MAL hits, steady-state backfill of bridge edges for pre-v0.14.1 rows AND pickup of MAL-added sequels into the catalog over the nightly window. Not counted as `anime_changed` for stable-counter purposes — edges are structural metadata, not the volatile canonical fields the counter tracks.
- **Title search: substring + pg_trgm bonus subtracted from cosine_distance.** "Lord of" should match "Lord of Mysteries" before "Overlord" even though embeddings think they're thematically close. Substring (ilike) is a flat bonus; pg_trgm `word_similarity()` is scaled linearly above a threshold so typos like "lor of" still surface the right show. Description/notes search unchanged (semantic, not literal). Required pg_trgm extension migration `4b8f1e3c7d0a`.
- **Anime-view filters mirror anime-card derivation, not per-media WHERE.** Filtering by "Currently Airing" against per-media `airing_status` surfaced animes whose only Finished side-story matched the WHERE clause but whose card showed "Currently Airing" via the priority-collapse rule. Filters now run against the same priority-collapsed value the card derives. Same for `age_rating` against MAX across media.
- **Merge survivor flow runs the classifier over A∪B.** Previously merge just re-parented B's media to A. Now the consolidated graph goes back through `reclassify_anime`, which can demote previously-Main media via the substance gate or re-anchor if B's main pre-dates A's. Admin sees the diff up-front via the "pending reclassifications" preview attached to each pending pair.
- **Chart-color remap for `alternative_version`.** Initial mapping put it on the `--ring` slot (light variant of `--primary`). Visually indistinguishable from main in default/blue themes (ΔL=0.07, Δhue=1.6°). Now mapped to yellow (palette[2]); summary stays on the secondary green; crossover absorbs the muted ring slot since it's the rarest type so the main↔crossover adjacency almost never happens.

## Failed Approaches

- **Filtering dangling edges at storage time.** Initial implementation only stored edges where the target was in the local catalog. Killed bridge-edge resolution at merge time. Reverted to unfiltered storage + adjacency-time filtering.
- **NULL-episodes-fails-substance-gate.** Demoted every currently-airing TV anchor until MAL filled the field. Backed off to "NULL episodes OK for currently_airing or null airing_status".
- **`get_first_main_relation` returning None → no anchor.** Broke donghua sub-universes whose canonical main is an ONA, and pilot-before-real-show franchises. Replaced with `_pick_root_for_promotion` that picks the most main-like graph entry by (type_tier, aired_from).
- **`relation_link` raw MAL strings.** Pre-v0.14.0 comparisons used the underscore form (`"main_story"`) but lists held raw MAL (`"Parent story"`) so the `is_main_story` gate silently let every franchise movie classify as `main`. Normalized once on entry via `_normalize_relation` — fix applies to new scrapes only, pre-fix rows keep stale labels until re-scraped via `relation_backfiller`.
- **AlternativeVersion → ring chart slot.** "Echo the main hue" intent was good UX in theory, indistinguishable in practice. Lesson: chart fills need lightness contrast, not hue family.

## Gotchas & Learnings

- **Pre-v0.14.1 rows missing `MediaRelationEdges` are the dominant cold-start path.** The seeder gates on `RELATION_BACKFILL_ON_STARTUP` so the ~14 min boot can be deferred on tight maintenance windows. Once primed, restarts touch zero rows.
- **`alternative_setting` is a graph boundary, not a relation type.** MAL uses it for "different story, shared themes" (typical bait for ML-style false positives). Treated as a graph cut like `crossover` AND dropped from the output entirely — including it as a relation_type triggered false-positive merge candidates on every sweep.
- **pg_trgm extension is a per-cluster install.** The `ankane/pgvector` Docker image bundles it; bare Postgres needs `CREATE EXTENSION pg_trgm` as superuser. CI bootstrap migration creates it before tests run.
- **Stable counter ignores edge churn.** Sweep step 1 writes `MediaRelationEdges` whenever the JSONB shape differs, but this does NOT set `anime_changed` — otherwise every MAL relation-list addition would reset the stability counter and drag every popular anime back to tier-1 cadence forever.

## Future Work / Known Follow-ups (surfaced by pre-PR review)

- **FMA inverse-merge.** Brotherhood is the canonical Main but aired AFTER the 2003 adaptation. Anchor-by-oldest-aired-from picks 2003 wrongly. Workaround: admin merges manually post-detection. Real fix: weight `scored_by` more heavily when within-tier ties are close in age (deferred to v0.15+).
- **Relation backfiller per-tier scheduling.** Currently rescans the whole catalog at every lifespan startup (cheap when sidecars are populated). A nightly tier-selected pass would let us drop the startup gate eventually.
- **pg_trgm GIN index + query shape.** `word_similarity()` in ORDER BY arithmetic (subtracted from `cosine_distance`) cannot use a GIN trigram index — Postgres must compute it per candidate row. Real fix is a query rewrite: pre-filter candidates via `col %> query` (where `%>` is the `word_similarity_op`), then the GIN index activates. Defer until search latency on `/search/anime` becomes a problem at catalog growth.
- **Merge-candidate preview recomputes classifier per pair on every admin page load.** For N pending pairs each `GET /admin/merge-candidates` runs `preview_reclassifications` (BFS + sort + closure) on the event loop. Acceptable at current pending count (<20), but linear in backlog. Options: wrap in `asyncio.to_thread()`, or cache the preview on the `merge_candidates` row at detection time and invalidate when either side's media set changes.
- **`vector_embedding_service.regenerate_anime_embedding` is DELETE-then-INSERT, not atomic.** Pre-existing. If the INSERT's encode call throws after the DELETE+flush, the row is gone with no replacement. Fix: encode both embeddings first, then DELETE + INSERT in that order. Now newly relevant because `relation_backfiller`'s per-anime try/except (added pre-PR) would commit a stale-deleted row if a caller catches the exception and continues.
- **Sweep step-1 silent-clobber on missing `/full` relations.** `parse_relation_edges(raw.get("relations") or [])` treats "MAL omitted the relations key" identically to "no relations" — zeroes out the sidecar. Fix: only update when `"relations" in raw`. Rare in practice (the existing `TransientUpstreamError` catches the obvious empty-payload case), but cheap to harden.
- **MAL rate-limit + base URL hardcoded** (`_MIN_REQUEST_INTERVAL_S=1.0`, `BASE_URL="https://api.jikan.moe/v4"`). Lift to `settings.JIKAN_MIN_REQUEST_INTERVAL_S` / `settings.JIKAN_BASE_URL` so a Jikan policy change or self-hosted Jikan mirror doesn't need an image rebuild.
- **`media_to_classifier_node` / `build_classifier_nodes` body duplication.** ORM-projection vs scraper-dict-projection helpers in `relation_classifier.py` emit the same `ClassifierNode` shape via two parallel constructors. Extract `_node_from_fields(media_type, aired_from, episodes, duration_seconds, scored_by)` shared by both. Currently colocated so the duplication is visible during edits, but a single helper would be safer when `ClassifierNode` gains fields.
- **`search_service.search_mal_api` reimplements `classify_and_stamp` body inline** (lines 71-74) because it also needs `nodes` afterwards (substance-gate check on the chosen anchor). Have `classify_and_stamp` return `(classifications, anchor, nodes)` so both call sites use the shared helper.
- **`reclassify_anime` 7-field copy** (drift detector + writer) refactored to dict-driven setattr loop pre-PR — adding an 8th umbrella field is now one edit.
- **`MaintenanceBanner` bypasses `api.ts` 503 handling** by design (raw `fetch` 30s poll). Pre-existing, called out by the reliability review as worth noting but not in this branch.
