## Summary

Bundled release: backend cleanup (UserDAO, domain exceptions, auth service as functions, CI test workflow) + full frontend migration to shadcn-svelte, Svelte 5 runes, centralized API client, TypeScript types, and a coherent theme system. License changed to PolyForm Noncommercial 1.0.0.

## Failed Approaches

- **shadcn Popover+Command for TagSelect**: UX was worse than inline combobox (separate search field, grey background, extra click). Reverted to custom inline approach with shadcn Badge for tags.
- **Standard light/dark theme**: The app's design (dark gradient background + light elevated surfaces) doesn't fit either mode. Required a custom split: `--background` dark, `--card`/`--popover` white, with `@custom-variant dark (&:is(.dark *))` to prevent OS dark mode interference.
- **`resolve.conditions` in vitest block only**: Svelte 5 requires `['browser']` at the top-level vite config, not just inside the test block, otherwise jsdom gets server-side Svelte and throws `component_api_invalid_new`.

## Key Decisions

- **Theme variables use `--color-` prefix** (Tailwind v4 requirement) not bare `--popover`. Missing prefix causes silent fallback to transparent.
- **`$app/state` over `$app/stores`** for Svelte 5 layout, but kept writable store for `token` since it needs `subscribe`/`set` from non-component code (api.ts, navigation.ts).
- **Single exception handler** with `status_code` class attribute on each exception, instead of per-exception handlers. Fewer lines, same behavior.
- **CI creates schema from `Base.metadata.create_all()`** instead of running alembic migrations, since migration files aren't committed.

## Gotchas

- Svelte 5 `onsubmit` does NOT auto-prevent default like Svelte 4's `on:submit|preventDefault`. Must call `e.preventDefault()` manually.
- `jwtDecode` throws `InvalidTokenError` on corrupt localStorage tokens. Without try/catch in the layout, user gets permanently stuck on loading screen.
- shadcn-svelte components import from `$lib/utils.js` (not `$lib/utils/index.js`), requiring a barrel file at `src/lib/utils.ts`.
