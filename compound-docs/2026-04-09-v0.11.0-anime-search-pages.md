# v0.11.0 — Anime Search & Detail Pages

**Date:** 2026-04-09 | **Branch:** v0.11.0-anime-search-pages

## Summary

Added anime-level aggregated search (two-phase GROUP BY + detail fetch), anime detail pages with bulk rating/delete, and ECharts-based ratings overview (quality radar, orbital attribute badges, relation-type timeline). View-type toggle lets users switch between anime and media search with view-aware filter options.

## Key Decisions

- **Two-phase aggregation query**: Phase A does GROUP BY with HAVING for filtering/ordering, Phase B fetches full ORM objects for matched IDs. Avoids computing aggregates in Python while preserving eager-loaded relationships. Trade-off: SQL aggregates computed in Phase A are discarded and re-derived in Python (Phase B needs additional fields like genres, seasons).
- **Genre majority filter via correlated subquery**: `genre_count * 2 > media_count` per genre in HAVING clause. Strict >50% majority semantic chosen over "any media has genre". Creates O(G) correlated subqueries per genre — acceptable for typical 1-3 genre filters but worth revisiting if filter UX allows many genres.
- **ECharts v5 pre-built bundle**: See `compound-docs/2026-04-06-echarts-integration.md` for full rationale. ~1MB bundle accepted because modular imports break under Vite's `resolve.conditions: ['browser']`.
- **Embedding backfiller generalized**: Single startup pass handles anime, media, and rating embeddings. Per-item error handling ensures one bad record doesn't block the entire backfill.

## Failed Approaches

- Tried ECharts modular imports and v6 — both failed due to tslib ESM resolution (documented in echarts compound doc).

## Gotchas & Learnings

- `selectinload` does not guarantee ordering — media list in anime detail needs explicit Python sort (by season year, season order, mal_id) for deterministic timeline display.
- PostgreSQL `func.log()` is log10, not natural log — easy to confuse in comments.
- INNER JOIN on `MediaSearch` for description search silently excludes media without embeddings. Changed to LEFT JOIN so `avg()` naturally ignores NULLs.
- AnyIO 4.1.0 renamed `cancellable=` to `abandon_on_cancel=` on `to_thread.run_sync`.

## Future Work / Technical Debt

- Optimize correlated genre subqueries (O(G) per anime group) if filter UX allows many simultaneous genre selections
- Revisit ECharts tree-shaking if Vite adds per-dep `resolve.conditions` overrides
- Carry Phase A SQL aggregates through to Phase B to avoid redundant Python recomputation
