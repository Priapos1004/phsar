## Key Decisions

- **Don't add a metric index for a global-rank query.** A partial expression index on `score * log(10, scored_by + 1)` was added for `score_top_percent`, then **removed after EXPLAIN**: ranking a row against *all* rows reads every scored row regardless, so the planner uses bitmap+sort even with `enable_seqscan=off` â€” an ordered index scan is never chosen (it returns ~all rows â†’ random I/O is worse than a sort). The index only added sweep write-overhead. The real lever at scale is caching percentiles at sweep time, not indexing.
- **`rank()` window form is not universally cheaper than `count(*) FILTER`.** For a single-row-vs-all rank, count-FILTER needs *no sort*; the window form forces one. So the two `score_top_percent` variants land on different optimal shapes by data shape: **anime â†’ window** (`rank() OVER (ORDER BY metric DESC) - 1`, because it removes a twice-referenced-CTE scalar subquery), **media â†’ count-FILTER** (already a single sort-free scan). Same logical result, deliberately divergent SQL.
- **The weighted-score formula spans two engines â€” lock them with a test, not a shared symbol.** SQL `weighted_score_expr` and Python `_weighted_score` (`math.log10`) must agree numerically. Made the SQL base explicit (`func.log(10, x)`, not Postgres's implicit single-arg base-10) and added a DB-backed equality test (`test_weighted_score_matches_python_twin`). A hybrid property was considered and **rejected**: it can't serve aggregate `avg()` inputs (anime variant) or old/new scalar pairs (sweep diff), so it'd be a *third* copy, not a unification.
- **Fix overstated copy at the copy layer.** The percentile tooltip read "higher than (100-N)%", but `score_top_percent` is floored at `max(1, â€¦)`, so the best title always claimed "higher than 99%" (nonsensical for tiny catalogs). Rephrased to the chip's own "Among the top N%" â€” the `max(1, â€¦)` floor is correct domain behavior ("Top 0%" is meaningless), so the bug was linguistic, not in the backend.

## Gotchas & learnings

- **Optional-helper fetch failure: leave state `null` + an error flag, never `[]`.** RatingCard's neighbor fetch set `neighborItems = []` on a transient 500, latching the misleading "you have no ratings to compare" empty state with no retry. Distinguish never-loaded (`null`) / failed (`null` + `neighborsError`) / genuinely-empty (`[]`) so a re-expand or a "Try again" button refetches.
- **Model-declared expression/partial indexes need `func`-over-columns, not bare `text()`.** A bare-`text()` `Index` has no column references, so SQLAlchemy can't attach it to the table â†’ `--autogenerate` reports it fully "removed". Mirror `ix_jobs_scrape_query` (`func.*` over real `Media.*` columns). Separately: Alembic can't round-trip *any* Postgres expression index (reflected `log(10::numeric,â€¦)::double precision` never string-matches the declared form), so they always show spurious churn that's curated out of generated migrations â€” the model declaration is the source of truth, the migration applies to existing DBs.
- **Rating-consistency pool is completed-only, from *other* anime, one row per anime.** Dropped scores are deflated and on-hold are provisional â€” mixing them would bias the very scale the helper steadies. Dedupe to one rating per anime (closest to score) so rows are distinct shows, not several seasons of one.
- **Any compact projection that renders a title must ship `name_eng` + `name_jap`, not just the canonical title.** The rating helper first shipped only `media_title`/`anime_title`, so rows ignored the user's name-language setting. Titles localize client-side via `resolveTitle` everywhere (detail pages, job-detail failure/probe lists), so the projection has to carry both alternate names for it to work.
- **Guard a single destructive icon-button with click-to-arm, not a modal.** The story-completion unmark (X) was a one-click `DELETE`. First click arms the row (X â†’ red Check, "click again to confirm", auto-disarms after ~3s); the second click within the window fires â€” the same confirm-step shape the merge/split candidate cards use. Clear the arm timer on unmount so it can't fire after navigation.
