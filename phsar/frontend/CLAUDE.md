# Frontend ‚Äî design notes

SvelteKit + Svelte 5 runes + Tailwind 4 + shadcn-svelte. Loaded on top of root [CLAUDE.md](../../CLAUDE.md) when working in the frontend tree.

## routes/

Pages: home (`/`), login (`/login`), register (`/register`), search (`/search`), media detail (`/media`), anime detail (`/anime`), settings (`/settings`), admin (`/admin`), add to library (`/library/add`).

- `GET /health` endpoint returning `{status, version}` for Coolify liveness ‚Äî deliberately does not probe backend (liveness should only check what restarting this container can fix)
- `/library/add` ‚Äî query-input + recent-additions panel; submitting POSTs to `/jobs/scrape` and the navbar bell takes over from there; restricted users see the page but the form is disabled
- **Tab title convention**: every route declares its own `<svelte:head><title>‚Ä¶</title></svelte:head>` in the format `<Page> ‚Äî Phsar` (em dash, "Phsar" title case). Page name first so users with several PHSAR tabs open can distinguish them in the tab strip; the favicon already conveys "this is PHSAR". Detail pages (anime/media) bind the title reactively to the resolved name and fall back to a generic label while loading. `app.html` only sets the default `PHSAR` so the very first paint isn't blank ‚Äî every route is expected to override
- **Admin tab navigation**: `/admin` uses a `?tab=` query param to switch between sections (`overview` default, `jobs`, `tokens`, `curation`, `completion`, `backups`). Tabs eager-render and stay mounted (visibility toggles via `class:hidden`), so first paint pays one parallel fetch per tab and subsequent switches are instant. The tab list, default, and validation live at the top of `routes/admin/+page.svelte`; the tab UI itself is `lib/components/admin/AdminTabNav.svelte`. Adding a tab requires updating three places: `AdminTabKey` in `lib/components/admin/types.ts`, the `TABS` array + content cascade in the page, and the rendering branch
- **Per-job detail page**: `/admin/jobs/[uuid]/+page.svelte` renders an update_sweep v2+ job's full audit trail ‚Äî header card, counters grid, filterable media-change list (chips: all / dynamic-no-rating / rating / static / has drift; search across all title-language fields) sorted most-substantial-first via `sortMediaChanges` (group static > genre/studio > dynamic > rating-only, then by amount changed; rating-only by search-relevance impact), umbrella "Anime changes" cards (rendered **above** Media changes since v0.14.9 ‚Äî far fewer rows, so they no longer get buried under 300+ media diffs). v4 (v0.14.7) renders a **Failed-refresh list card** (per-anime `step1_failures`); v5 (v0.14.8) adds a symmetric **Failed-probe list card** (`probe_failures` ‚Äî title link + `error_category` + message) and a **progress-divergence warning** when `items_done < items_total` (anime selected but skipped); v6 (v0.14.9) adds an **"Attached via probe" card** (`probe_attached_anime` ‚Äî anime title link + the new media titles listed below, each linked to its media detail). The three list cards (failed-refresh / failed-probe / attached) cap at ~4 rows and scroll (`max-h-72 overflow-y-auto`) so a long list doesn't bury the changes below. **Link-target design decision (v0.14.9):** the failure + attached links navigate in the **same tab** with a `from: 'job'` origin (‚Üí "Back to job" button, propagated across anime‚Üîmedia hops) ‚Äî short lists, no scroll to lose; the **Anime/Media change** cards (`AnimeUmbrellaCard` / `MediaChangeCard`) instead open a **new tab** with no origin, because that list runs 300+ rows and an admin verifying one row shouldn't lose their scroll position. Failed jobs only show the error banner in the header; v1 sweeps render a "predates v0.14.5" notice. Click-through from `AdminJobsLogTab` is restricted to `update_sweep && version >= 2`; its `payloadSummary()` reads media-grained keys (`media_refreshed`, `media_with_dynamic_changes`) for v5 and the anime-grained keys for v2‚Äìv4
- **Settings page** (`/settings`): restricted (guest) users see **all controls disabled, not hidden** ‚Äî Rating Step, Data Export buttons, and the Spoiler Protection select (pinned to `Off`, since guests can't rate and are excluded from the spoiler cache). Theme / name-language / search-view stay editable. The server also drops a guest's `spoiler_level` update defensively (`user_settings_service`)
- **Detail-page back button**: anime/media routes render `<BackLink searchToken fromParam jobUuid />` near the top. It decides between "Back to search" (when `?q=<token>` is present, from search results), "Back to library" (when `?from=library`, e.g. recent-additions panel), "Back to job" (when `?from=job&job=<uuid>`, from the admin job-detail page's Failed-refresh / Failed-probe / Attached-via-probe links ‚Üí `/admin/jobs/[uuid]`), "Back to completion" (when `?from=completion`, from the admin Completion tab's anime links ‚Üí `/admin?tab=completion`), and "Back to curation" (when `?from=curation`, from the Merge/Split candidate cards' anime links ‚Üí `/admin?tab=curation`). New origins extend the `DetailOrigin` type union in `lib/utils/navigation.ts` AND the `target` switch in `BackLink.svelte`. `buildDetailHref()` propagates `q`/`from`/`job` on internal anime‚Üîmedia jumps (incl. the RelatedMediaCarousel) so a deep dive (job ‚Üí anime ‚Üí media ‚Üí sibling) stays linkable back to the origin

## lib/components/

App components using Svelte 5 `$props()`, `$state()`, `$derived()`, `$effect()`.

- **SearchBar, MediaInfo, NavBar, TagSelect, DoubleRangeSlider, RatingCard, BulkRateDialog, DangerZone, RelatedMediaCarousel, SpoilerGuard, EChart** ‚Äî core UI components
- **Tooltip** ‚Äî the single app-wide hover/focus tooltip (wraps the shadcn-svelte/bits-ui `ui/tooltip` primitive). Themed to the active palette: a faint primary-tinted `popover` surface with a `--primary` border + soft primary glow (see `ui/tooltip/tooltip-content.svelte`), ~500ms delay. **Self-contained** ‚Äî it renders its own `Tooltip.Provider`, so it works standalone (incl. isolated component tests) with no ancestor Provider. Two modes: pass `children` for non-interactive hints (renders a `cursor-help` span trigger ‚Äî status dots, labels, chart bars) or a `trigger` snippet for interactive elements (spread `props` onto your own `<Button>`/`<button>` so keyboard focus reveals it and nothing gets double-wrapped; icon-only buttons must add an `aria-label`). Note: a test that mocks `svelte`'s `setContext`/`getContext` must delegate unknown keys to the real impl, or the Provider context breaks (see `tests/media-detail.test.ts`)
  - **The three tooltip types** (pick by what the cursor is over ‚Äî never style raw text as a hint):
    1. **App `Tooltip` component** (`lib/components/Tooltip.svelte`) ‚Äî the themed DOM hint, for every intentional UI hint on a DOM element. Examples: `GenreBadges` (description on a badge), `ScorePercentile` (the "Top N%" pill), `RatingsFilterBar` grid-order toggle (icon button ‚Üí `trigger` snippet + `aria-label`), `JobDetailCounters` cells
    2. **ECharts chart tooltip** ‚Äî the in-canvas hover for any `EChart`-based chart, configured on the chart's `option.tooltip` (the app `Tooltip` can't wrap a canvas region, so charts own their hover). Spread the shared **`chartTooltipStyle`** from `lib/utils/chartTheme.ts` into every chart's `tooltip` so the surface/border/glow match the app `Tooltip` (it themes via CSS custom props ‚Äî `--tooltip-surface`, `--primary` ‚Äî which resolve per active theme since the ECharts tooltip is a DOM node, and bundles `confine: true` so the popup stays in the card), then add the chart's own `trigger` / `formatter`. Examples: `RatingsOverviewTimeline.svelte` (anime page, per-media score bars) and `ratings/RatingsScoreHistogram.svelte` / `RatingsAlignmentChart.svelte` (ratings stats). Spreading `chartTooltipStyle` is the single source ‚Äî don't re-set its keys per chart
    3. **Native `title=`** ‚Äî raw text-overflow reveal on dense data cells ONLY (`MediaChangeCard` / `AnimeUmbrellaCard` / search-result titles / `RatedAnimeCard` title) so they don't each become a keyboard tab stop or spin up a Provider per row
- **GenreBadges / StudioLinks / ScorePercentile** (v0.14.11) ‚Äî shared detail-page chips extracted from the anime+media `+page.svelte` (both rendered identical inline markup; extraction de-duplicates and makes them testable):
  - **GenreBadges** (`genres: string[]`) ‚Äî renders each genre as a `cls.badgeGenre` Badge; wraps it in the app `Tooltip` with the genre's description when one exists (else a plain badge). Descriptions come from the `genres` store (lazy-fetched once); the tooltip is a progressive enhancement, so a missing description or failed fetch just drops the tooltip
  - **StudioLinks** (`studios: string[]`) ‚Äî renders the "Studio" label + one clickable `<button>` badge per studio; clicking calls `navigateToSearch({ query:'', search_type:'title', view_type:'anime', studio_name:[studio] })` so the user lands on an anime-view search filtered to that studio ("other anime from this studio"). Tested in `tests/studio-links.test.ts`
  - **ScorePercentile** (`topPercent: number \| null`) ‚Äî small always-visible "Top N%" pill placed **after** the score's rating-count text (raw MAL data first, then the derived standing), color-ramped by tier (‚â§15% emerald / ‚â§50% sky / else neutral). Always-visible so mobile doesn't depend on hover; the `Tooltip` only adds the full "among the top N% ‚Ä¶ weighted by vote count" explanation (phrased as the clamped chip rank, not an inverted "higher than (100-N)%" figure ‚Äî the latter overstated the best title since `topPercent` is floored at 1). `null` renders nothing
- **RatingsOverview** ‚Äî with sub-components: Stats, Timeline, Notes, Attributes
- **DeleteWatchHistoryToggle** (v0.14.10) ‚Äî shared opt-in "Also delete watch history" checkbox used by both the single-rating delete dialog (RatingCard) and the bulk-delete dialog (anime page). The standing explanation lives in the component; callers pass only the count sentence (`detail`) so the copy can't drift
- **RatingCard rating-consistency helper** (v0.14.11) ‚Äî a collapsible "How you rated similar titles" panel inside the edit form. On first expand it fetches `GET /ratings/scores` once and caches it; a `$derived` over the live `snappedScore` runs `selectRatingNeighbors` (utils) to pick the 2 closest at-or-below + 2 closest above the current score, each from a **distinct other anime** ‚Äî so moving the slider re-selects instantly with no refetch. A failed fetch leaves `neighborItems` null (not `[]`) and sets `neighborsError`, rendering a "Couldn't load ‚Äî Try again" state instead of the misleading empty-state copy, so a re-expand or the retry button refetches. Rows expand to show that title's attributes. The media page passes the current media's `animeUuid`/`genres`/`studios`/`ageRatingNumeric` (optional props) as the exclude + tiebreak context
- **MediaInfo `is_finished` badge** (v0.14.10) ‚Äî anime search cards render a small emerald "‚úì Complete" badge by the title when the anime is story-complete (anime cards only, never media). **No tooltip in search** ‚Äî hover popups while scrolling are intrusive; the full airing-vs-story explanation lives only on the anime detail page's "Story Complete" badge (app `Tooltip`)
- **AttributeRadar, AttributeBadges, AttributeDetailBars** ‚Äî attribute visualization
- **VersionFooter** ‚Äî renders at the bottom of every page, reads `PUBLIC_APP_VERSION` from `$env/dynamic/public`
- **LoadingScreen** ‚Äî themed sakura-ring loader shown during initial boot + ~1.5s logout transition
- **Notice** ‚Äî shared yellow info card (rounded `bg-yellow-50` surface + `AlertTriangle` icon ‚Äî solid surface so it reads on the dark body gradient)
- **BackLink** ‚Äî shared back-button used by anime/media detail pages; renders nothing when neither `searchToken` nor a known `fromParam` is set (so direct-URL arrivals stay clean)
- **RelatedMediaCarousel** ‚Äî sibling-media row on the media detail page. Backend (`media_search_service.get_media_detail`) returns `sibling_media` chronologically via the shared `chronological_media_key()` helper in `filter_service.py` ‚Äî same key the spoiler frontier and anime-detail media table use ‚Äî and a `current_position` insertion index. The carousel renders a "You are here" divider at that slot; frontend doesn't compare dates, the backend owns the ordering
- **BackupsCard** ‚Äî admin-only dump list
  - Create/upload/download/restore/delete with a "Current" badge on the row the DB was last restored from
  - **Rename / pin** (pencil ‚Üí inline `Input` + Save/Cancel; `PATCH /admin/backups/{filename}`): a non-empty name pins the dump against auto-retention and renders a "Pinned" badge. Pinned rows show a one-click `PinOff` action directly in the row (no need to enter edit mode); saving a blank name in the editor also unpins. `saveName(filename, name)` is the shared call behind `handleRename` (Save) and `handleUnpin` (one-click remove)
  - Refreshes are silent ‚Äî `refresh()` flips `loading` only on the initial mount, so a refetch after rename/unpin/delete/restore diffs the keyed `{#each}` in place instead of collapsing to a spinner and snapping scroll to the top (same pattern as MergeCandidatesCard)
  - **Restore link**: the `is_current` row shows "Previous state saved as `{previous_state}`"; a `pre_restore` row shows "Snapshot of the state before restoring `{restored_to}`" ‚Äî both derived from sidecar metadata, no client computation
- **MergeCandidatesCard** / **SplitCandidatesCard** ‚Äî admin-only review surfaces for pending merge/split candidates
  - Side-by-side anime info (with rating count + earliest aired date as visible justification for recommended A)
  - Similarity score, merge/dismiss with confirm-step, per-row "Swap A/B" button
  - Anime titles link **same-tab** with `from=curation` (‚Üí "Back to curation" ‚Üí `/admin?tab=curation`), via `buildDetailHref` ‚Äî matches the Completion tab (v0.14.11; replaced the old `target="_blank"` so a verifying admin returns to the same list instead of tab-juggling)
  - Silent refreshes ‚Äî keyed each-block diffs in place instead of collapsing to "Loading‚Ä¶"
  - Merges trigger automatic refetch surfacing cascade-resolved pairs and freshly-detected ones
  - Both embed **admin/DismissedDecisionsSection** at the bottom (a "Dismissed decisions" expander; see below), passing `currentUsername` (threaded from the admin page's `getUsername()` context) and `onResurfaced={handleRedetect}`
- **admin/CompletionStatusCard** ‚Äî the **Completion** tab (v0.14.10). A manual curation surface (not a detection queue): a debounced anime search (reuses `fetchAnimeSearchResults` ‚Üí `/search/anime`, with a clear-‚úï button) to mark anime story-complete, plus the marked list (cover thumbnail + "marked {date} by {admin}" audit, client-side sort newest/oldest/title, unmark). Marking clears + closes the search so it reads as a committed selection. Same silent-refresh pattern as MergeCandidatesCard. Anime links navigate same-tab with `from=completion` (‚Üí "Back to completion"). Unmark (X) is guarded by a **click-to-arm confirm** (v0.14.11): first click arms the row (X ‚Üí red `Check`, tooltip "Click again to confirm removal", auto-disarms after 3s), the second click within the window fires the `DELETE` ‚Äî mirrors the candidate cards' confirm-step so an accidental click can't silently un-mark. No `currentUsername` prop ‚Äî the admin role gates the page
- **admin/DismissedDecisionsSection** ‚Äî shared, generic (`<T extends {uuid, dismissed_at}>`) collapsible history of dismissed merge/split decisions, reused by both candidate cards. Lazy-fetches its `listUrl` on first expand; renders each row via a parent-supplied `row` snippet (merge shows A‚ÜîB; split shows the source + "would split off:" member titles per cluster). A username-gated confirm dialog (mirrors BackupsCard restore) POSTs `${basePath}/{uuid}/delete`, then calls `onResurfaced()` so the parent re-runs detection and the freed candidate pops back into the pending list. Subscribes to `curationRefresh` (`onBump`) so the counter/list re-fetch when the parent dismisses a new candidate ‚Äî no stale `(N)`
- **MaintenanceBanner** ‚Äî wraps a `Notice`, lives in a sticky container in `+layout.svelte` alongside the navbar
  - Polls `GET /maintenance/status` every 30s via raw `fetch` (bypasses `api.ts` so a 503 mid-window can't trigger maintenance redirect and defeat the pre-warning point ‚Äî 30s instead of 60s because seasonal sweep's maintenance window can be only seconds long)
  - Subscribes to `token` store + `maintenanceRefresh` bump signal so relogin or any 503-with-maintenance response refreshes in milliseconds
  - Countdown wording: `"Scheduled maintenance starts in ~N minute(s) ‚Äî pause your current episode."` within 30 min; `"Maintenance in progress. Please try again later."` when active; otherwise hidden
- **JobBell** ‚Äî polls `/jobs/mine`
  - Caps dropdown at 5 entries (older spill to `/library/add`)
  - Hides retry button when `result_summary.retryable === false`
  - Disables retry across all rows while one is in flight
  - Progress bar uses `Math.min(100, Math.round(percentOf(items_done, items_total)))` ‚Äî backends occasionally over-report a child batch as more items than the initial total estimate; the 100-cap keeps the bar from rendering 137%
- **admin/AdminJobsLogTab** ‚Äî paginated jobs table with kind/status filters, seasonal_sweep child expander
  - Kind/status filters persist via the `jobsFilter` module store (`lib/stores/adminJobsFilter.ts`), **not** the URL. The store is in-SPA memory that survives admin tab switches AND the round-trip through `/admin/jobs/[uuid]` ‚Äî both navigate to a bare `/admin?tab=jobs`, and a mounted module store carries the filter through without re-threading it (so the detail back link stays a plain `/admin?tab=jobs`). The tab reads the store via `$derived` (drives the Select + the fetch) and writes it in the filter setters; a `$effect` on `($jobsFilter, offset)` re-fetches. `routes/admin/+layout.svelte` cascades over the whole section and calls `clearJobsFilter()` `onDestroy`, so re-entering /admin from elsewhere (e.g. Settings) starts clean while in-section hops keep the filter. Deliberately **no** URL mirror: it's an internal tool, refresh-survival/shareable-links aren't worth the reconcile complexity, and a URL copy would resurrect a filter on browser-back after leaving. Values are whitelisted against `JOB_KIND_LABELS` / `STATUS_BADGE` keys (`sanitizeKind`/`sanitizeStatus`); pagination offset stays in component state. Helpers unit-tested in `tests/admin-jobs-filter.test.ts`
  - Live-ticking Duration column for `running` rows (shared `formatJobDuration` + a `hasRunning` short-circuit derived so the 1s interval is quiet when nothing is in flight)
  - Click-through to `/admin/jobs/[uuid]` ONLY for `update_sweep && version >= 2` rows ‚Äî the detail page has nothing to add for other kinds, so the row stays non-clickable (no cursor, no hover, no keyboard handler) and the click affordance matches what's behind it
  - Amber row tint + inline "‚öÝ N unknown genre tags need seeding" subline when `result_summary.unknown_genre_tags` is non-empty (v3 update_sweep only), so the admin spots seeder-update work without drilling into the detail page. v6 (v0.14.9) adds a parallel **blue** tint + "üîó N media added via probing" subline keyed on `counters.probe_attached_media_count > 0` (amber takes precedence when a row has both); media-grained, so pre-v6 rows stay un-tinted
- **admin/SweepTiersCard** ‚Äî admin Overview card showing where every row sits in the update-sweep cycle, with an **anime/media toggle** (v0.14.8, default anime). Takes `animeTiers` + `mediaTiers`; backend exposes 4 mutually-exclusive **cycle-membership** priority-cascade buckets (airing_now / stabilizing / weekly_cycle / long_cycle) at each grain via `sweep_tiers` + `media_sweep_tiers`; the card renders the selected grain 1:1 (no fold). Membership not due-ness ‚Äî counts stay stable across sweeps instead of emptying when a sweep refreshes a tier. Per-row horizontal share bar, count, percentage, tooltip with the underlying predicate (stabilize < 3 sweeps, 90-day long cycle ‚Äî both shared by the two grains). The Stabilizing row expands into per-check **sub-rows** (`stabilizing_by_check` ‚Äî one per `stable_check_count` 0‚Ä¶threshold-1) so the stabilization pipeline is visible; bar opacity ramps with the check count (closer to graduating = more filled). Media grain buckets each media by its own count; anime grain by its **least-settled member** (MIN across its media). Sub-row count is derived dynamically (`stabilizeThreshold = Object.keys(stabilizing_by_check).length`) so retuning `SWEEP_STABILIZE_THRESHOLD` reshapes the card with no frontend edit
- **admin/JobDetailHeader / JobDetailCounters / MediaChangeCard / AnimeUmbrellaCard** ‚Äî sub-components of the `/admin/jobs/[uuid]` detail page
  - `MediaChangeCard`: 5 tones (rating-zinc / dynamic-amber / static-sky / genre-fuchsia / studio-indigo) with a colored left border per row; rating sorts last so vote-count churn doesn't drown the actionable bits. Genre / studio drift folds into the same Field/Was/Now table ‚Äî was-cell shows old tags with removed in red, now-cell shows new tags with added in green; v2 (legacy) drift kinds render a "Not auto-applied" subline since the dispatcher only logged them
  - `AnimeUmbrellaCard`: per-anime 7-field umbrella diff + per-media relation reclassifications + "anchor moved" / "embedding regen" badges
  - `JobDetailCounters`: takes `version`; the STATS array is version-aware ‚Äî v5+ shows media-grained counters (`media_refreshed`, `anime_touched`, `media_skipped_fresh`) and drops the `anime_with_*` rollups, v2‚Äìv4 show the anime-grained originals. Most cells carry an explanatory `Tooltip`. v0.14.9 **dropped the `step1_failed` / `probe_failed` cells** (the amber "Failed refresh" / "Failed probe" cards below already carry the count, so a grid cell just duplicated the signal ‚Äî and with them gone the version-gated amber-warn logic went too), and softened labels: "Anime reclassed" ‚Üí "Anime changed", "Anime w/ new attach" ‚Üí "Anime w/ new media"

## lib/

- **api.ts** ‚Äî centralized API client
  - Methods: `get`/`post`/`postForm`/`put`/`del`/`downloadBlob`/`postMultipart`
  - `ApiError` class, automatic auth header injection from token store
  - Maintenance-503 handler: clears token, calls `bumpMaintenanceRefresh()` (covers user-on-/login where `token.set(null)` is a no-op), hard-navigates to `/login`
- **types/api.ts** ‚Äî TypeScript interfaces mirroring backend Pydantic schemas (`MediaConnected`, `FilterOptions`, `TokenResponse`, etc.)
- **stores/** ‚Äî Svelte stores for auth, settings, spoiler visibility, bell session, cross-component bumps
  - `jobs.ts`:
    - Three writable bumps via `createBumpStore()` factory + `onBump(store, fn)` helper:
      - `jobsRefresh` ‚Äî bumped by `/library/add` or BackupsCard after successful POST so bell refetches in ms instead of waiting for 30s idle poll
      - `librarySaved` ‚Äî bumped by bell when it observes a new `succeeded` `user_scrape` so `/library/add` refreshes recent-additions
      - `backupSaved` ‚Äî bumped by bell on new `succeeded` `backup` so BackupsCard auto-refreshes
    - Same `announcedSavedUuids` Set covers both kinds ‚Äî UUIDs are globally unique
    - `optimisticJobs` (writable list) + `addOptimisticJob(job)` / `reconcileOptimisticJobs(fetched)`:
      - Bell merges optimistic ‚à™ fetched (deduped by UUID) inside `$derived`
      - Callers seed a `queued` row synchronously off returned `job_uuid`; next fetch reconciles (UUID match ‚Üí optimistic pruned)
    - Store-update helpers short-circuit no-ops (svelte writables notify on every `set()` even with same-reference returns)
  - `maintenance.ts` ‚Äî `maintenanceRefresh` writable + `bumpMaintenanceRefresh()`; `api.ts` bumps on every 503-with-maintenance, `MaintenanceBanner` subscribes via `onBump`
  - `bell-session.ts` ‚Äî `BELL_LOGIN_KEY` / `BELL_SEEN_KEY` + `clearBellSession()` helper; `auth.ts` calls it when token transitions to null so logout-and-back-in doesn't inherit previous session state (sessionStorage survives hard navs)
  - `genres.ts` (v0.14.11) ‚Äî `genreDescriptions` map store (lowercased name ‚Üí description) + `ensureGenresLoaded()`. Lazy-fetches `GET /filters/genres` once per session (concurrent callers share one in-flight promise) and caches the lookup for GenreBadges' tooltips; a failed fetch clears the promise so a later mount retries
- **utils/** ‚Äî helpers:
  - String formatting: `formatAiringStatus`, `formatRelationType`, `formatMediaType`, `formatSeasonRange`, `formatDuration`, `formatJobDuration`, `formatDecimalDigits`, `formatShortDate`, `formatShortDateTime`, `formatBytes`
  - Numeric: `percentOf(value, total)` ‚Äî share-of-total with divide-by-zero guard; callers compose with `Math.round` (integer %) or `.toFixed(1)` (decimal %). `isRatingField(name)` ‚Äî single source of truth for the score / scored_by sub-bucket split used by `MediaChangeCard.dynamicTone` and the route page filter chip
  - Season logic, search params (`fetchSearchResults`, `fetchAnimeSearchResults`), navigation (`navigateToSearch`, `buildDetailHref`)
  - Chart colors: `CHART_COLORS`, `scoreColor`, `getThemedChartColorPalette`, `RELATION_TYPE_ORDER`, `RELATION_TYPE_COLORS`, `RELATION_TYPE_LABELS`
  - `jobBadges.ts` ‚Äî `STATUS_BADGE: Record<JobStatus, string>` shared between `AdminJobsLogTab` and `JobDetailHeader` so status pill colors can't drift
  - `spoilerFrontier.ts` ‚Äî client-side frontier for detail pages
  - `ratingNeighbors.ts` ‚Äî `selectRatingNeighbors(items, score, ctx)` for the RatingCard consistency helper: pool is **completed** ratings from **other** anime (dropped/on-hold excluded ‚Äî biased/provisional scores), deduped to one rating per anime (closest to score); returns the 2 closest at-or-below + 2 closest above, tiebroken by attribute-count ‚Üí shared-genre ‚Üí shared-studio ‚Üí age-distance ‚Üí recency. Pure so it recomputes per slider move and is unit-tested (`rating-neighbors.test.ts`)
  - `mediaChangeSort.ts` ‚Äî `sortMediaChanges` for the job-detail media-change list (category-priority then amount-changed; rating-only by weighted `score*log10(scored_by+1)` delta ‚Äî matches the backend search ranking). Decorate-sort-undecorate so the per-row key is computed once per re-sort; extracted from the page so it's unit-testable
- **themes.ts** ‚Äî centralized theme config: `THEMES` record mapping keys to CSS classes, character pics, labels; `ThemeKey` type; helpers: `isValidTheme`, `getThemeCssClass`, `getThemePic`, `getThemeFocal`, `getActiveTheme`
- **echarts.ts** ‚Äî lazy-loaded ECharts singleton (`getEcharts()`) using pre-built ESM bundle (SSR-safe, cached)
- **config.ts** ‚Äî backend API base URL (consumed only by `api.ts`)

## src/app.css

Theme system: `@property` definitions for `--primary`/`--ring`, `@theme inline` with `var()` indirection, `.theme-red`/`.theme-blue`/`.theme-green` override classes, themeable gradient variables (body dark gradient `--gradient-*` + auth-page light gradient `--auth-gradient-*`). Light elevated surfaces on dark gradient background. Dark mode locked to class-based only.

## tests/

Vitest + @testing-library/svelte component tests.
